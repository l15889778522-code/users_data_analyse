# Instagram 用户行为分析 SQL 项目

这是一个基于 MySQL 的用户行为分析项目，使用一个类 Instagram 社交平台数据集，对用户增长、内容生产、互动行为、关系网络和用户分层进行分析，并产出可复用的 SQL 查询与业务报告。

## 项目亮点

- 使用 MySQL CTE、聚合、条件分层、窗口函数和多表关联完成用户行为分析。
- 从用户、照片、点赞、评论、关注、标签等表中构建完整分析链路。
- 识别核心业务问题：平台互动密度高，但内容供给不足。
- 输出可直接用于简历展示的分析报告、SQL 查询文档和复现脚本。

## 数据表

核心数据来自 `instagram` 数据库：

| 表名 | 含义 |
| --- | --- |
| `users` | 用户注册信息 |
| `photos` | 用户发布照片 |
| `likes` | 用户点赞行为 |
| `comments` | 用户评论行为 |
| `follows` | 用户关注关系 |
| `tags` | 标签字典 |
| `photo_tags` | 照片与标签关系 |

## 核心结论

- 数据集中共有 100 个用户、257 张照片、8,782 次点赞、7,488 条评论和 7,623 条关注。
- 77% 的用户属于重度主动互动层，说明社区互动意愿较强。
- 26% 的用户没有发布任何照片，13% 的用户没有任何主动行为。
- 平均每张照片获得 34.17 次点赞和 29.14 条评论，内容一旦发布就能获得较高互动。
- 用户关系网络非常均匀，每个用户收到 76-77 个关注，主要短板不是连接关系，而是内容生产。

## 业务建议

1. 优先激活零发图和低发图用户，降低首次发布门槛。
2. 将高互动但低产出的用户转化为内容创作者。
3. 利用热门标签优化发布提示和内容推荐。
4. 补充真实行为时间线，用于后续留存、周报和趋势分析。

## 如何复现

1. 启动 MySQL，并确保本地存在 `instagram` 数据库。
2. 连接数据库：

```powershell
& 'H:\MySQL\MySQL Server 8.0\bin\mysql.exe' -h localhost -P 3306 -u root -p
```

3. 切换数据库：

```sql
USE instagram;
```

4. 执行 SQL 脚本：

```powershell
& 'H:\MySQL\MySQL Server 8.0\bin\mysql.exe' -h localhost -P 3306 -u root -p instagram < sql\instagram_user_analysis.sql
```
